Install & Use the Paraloom Plugin

Claude Desktop · about 5 minutes · no terminal needed

Before you start: you need Claude Desktop on a paid plan (Pro, Max, Team, or Enterprise) and a Paraloom login. Download the plugin file paraloom-plugin.zip from github.com/metrifi/paraloom-plugin (in the dist folder) and save it where you can find it.

Install in 7 steps

  1. Open Claude Desktop and go to Settings
    Click your name or the gear icon to open Settings.
  2. Open the Plugins section
  3. Add → Upload plugin
    Click Add, then choose “Upload plugin.”
  4. Drop in paraloom-plugin.zip and click Upload
    Select the zip you downloaded. It installs as “ready to use.”
  5. Open the Connectors tab and click Install next to Paraloom
    This adds the Paraloom connector. The button then changes to Connect.
  6. Click Connect
    The Paraloom sign-in window opens.
  7. Log in to Paraloom
    After you sign in, Claude Desktop shows Connected. You only do this once.
You’re connected. Start a new chat and try one of the prompts on the next page. If a chat you already had open doesn’t see the plugin, fully quit Claude (Cmd‑Q) and reopen it.

Using the plugin

The plugin runs the Paraloom experiment workflow. Just type one of these in a normal chat and swap in your own details. It handles the phases and pauses at the points that need a human.

Kick off an experiment
Run an experiment for [credit union] on [topic].

Example: “Run an experiment for Listerhill Credit Union on first-time homebuyer mortgages.” It researches real search demand, designs the experiment, drafts the article, and runs the compliance/fact/accessibility reviews — then prepares the client deliverable and stops for the FI sign-off.

Send it to the customer
Send the [experiment] deliverable to the client.

It builds and publishes the deliverable and drafts the client email — then waits for your OK before anything is actually sent.

Check for revisions
Check [experiment] for client responses and apply any revisions.

It pulls the client’s answers and checklist confirmations, applies them the right way, re-runs the reviews, and pushes an updated version. If nothing new came in, it says so and stops.
